# Test Plan – Binance QA

## Objective

Validate core user journey:

* Registration
* KYC
* Funding
* Trading

## Scope

In Scope:

* Functional testing
* UI validation
* Basic workflow testing

Out of Scope:

* Performance testing (JMeter)
* Security penetration testing

## Test Approach

* Manual testing
* Positive & negative scenarios
* End-to-end validation

## Entry Criteria

* Application accessible
* Test data available

## Exit Criteria

* All critical test cases executed
* No high severity defects open

## Risks

* Payment gateway dependency
* KYC approval delays
* Market price fluctuation

## Deliverables

* Test Cases (Excel)
* Execution Report
* Defect Logs
